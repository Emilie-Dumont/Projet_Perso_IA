# Cahier des Charges — NarrativePath (version finale)

## 1. Problématique

Dans un contexte réel, un parent souhaite proposer une histoire interactive à son enfant via une tablette.

Aujourd’hui, deux solutions existent :
- Les livres interactifs : contenu limité et répétitif
- Les outils IA (type AI Dungeon) : contenus non contrôlés

**Problème :**
Comment proposer une narration interactive infinie, personnalisée et sécurisée pour des enfants (6–12 ans) et des adolescents (13–18 ans) ?

---

## 2. Objectif du projet

Créer une application web de fiction interactive :

- accessible immédiatement (sans compte)
- sécurisée par design
- personnalisée (prénom, style, âge)
- basée sur des choix narratifs

---

## 3. Fonctionnalités principales

### 🔹 Moteur narratif
- système de nœuds
- chaque nœud contient :
  - un texte
  - 2 à 3 choix
- navigation entre les scènes

### 🔹 Personnalisation
- prénom de l’utilisateur intégré dans l’histoire
- choix du style narratif :
  - aventure
  - fantastique
  - humour
- adaptation du contenu selon l’âge :
  - 6–12 ans
  - 13–18 ans

### 🔹 Sécurité
- contenu adapté à la tranche d’âge
- simulation de prompts IA sécurisés
- filtrage par design (et non après génération)

### 🔹 Rejouabilité
- bouton "Recommencer"
- possibilité de tester plusieurs parcours

---

## 4. Architecture technique

- Vite.js (application frontend)
- JavaScript Vanilla
- Tailwind CSS (UI)
- JSON mock pour les données
- localStorage pour sauvegarde
- aucune API externe
- aucune base de données
